**Cart To Order Trace**

The main product-to-order path is:

`RouteProvider` -> `ShoppingCartController` -> `ShoppingCartService` -> `CheckoutController` -> `OrderProcessingService.PlaceOrderAsync` -> `OrderService` repositories -> `ShoppingCartService.ClearShoppingCartAsync`.

Key evidence:

1. Cart entry points are AJAX routes in [RouteProvider.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Presentation/Nop.Web/Infrastructure/RouteProvider.cs:201):
`addproducttocart/catalog/...` maps to `ShoppingCart.AddProductToCart_Catalog`, and `addproducttocart/details/...` maps to `ShoppingCart.AddProductToCart_Details`.

2. Catalog add loads the customer/store cart, validates, then calls `_shoppingCartService.AddToCartAsync(...)` in [ShoppingCartController.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Presentation/Nop.Web/Controllers/ShoppingCartController.cs:593). Product-detail add parses quantity, attributes, rental dates, then calls `SaveItemAsync` in [ShoppingCartController.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Presentation/Nop.Web/Controllers/ShoppingCartController.cs:791).

3. `SaveItemAsync` chooses the cart mutation:
new item -> `_shoppingCartService.AddToCartAsync(...)`; edit -> `_shoppingCartService.UpdateShoppingCartItemAsync(...)`; duplicate merge -> delete other cart item. See [ShoppingCartController.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Presentation/Nop.Web/Controllers/ShoppingCartController.cs:305).

4. `ShoppingCartService.AddToCartAsync` is the persistence point for new cart rows. It resets checkout data, loads the customerâ€™s cart, finds an equivalent line, then either updates quantity via `_sciRepository.UpdateAsync(shoppingCartItem)` or creates a new `ShoppingCartItem` and calls `_sciRepository.InsertAsync(shoppingCartItem)`. See [ShoppingCartService.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Libraries/Nop.Services/Orders/ShoppingCartService.cs:1549).

5. Checkout confirmation reloads the persisted cart with `_shoppingCartService.GetShoppingCartAsync(customer, ShoppingCartType.ShoppingCart, store.Id)`. Standard checkout does this in `ConfirmOrder`, then builds/stores a `ProcessPaymentRequest` and calls `_orderProcessingService.PlaceOrderAsync(processPaymentRequest)`. See [CheckoutController.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Presentation/Nop.Web/Controllers/CheckoutController.cs:1286).

6. One-page checkout converges on the same call: `OpcConfirmOrder` validates the cart, prepares the same `ProcessPaymentRequest`, then calls `_orderProcessingService.PlaceOrderAsync(processPaymentRequest)`. See [CheckoutController.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Presentation/Nop.Web/Controllers/CheckoutController.cs:2019).

7. `OrderProcessingService.PlaceOrderAsync` prepares details, processes payment, saves the order header, moves cart lines to order items, saves discounts/gift-card usage, sends notifications, resets checkout data, publishes `OrderPlacedEvent`, and checks order status. See [OrderProcessingService.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Libraries/Nop.Services/Orders/OrderProcessingService.cs:1567).

8. During preparation, `PrepareAndValidateShoppingCartAndCheckoutAttributesAsync` reloads the authoritative shopping cart into `details.Cart` with `_shoppingCartService.GetShoppingCartAsync(...)`, validates the cart, and validates every `ShoppingCartItem`. See [OrderProcessingService.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Libraries/Nop.Services/Orders/OrderProcessingService.cs:483).

9. `SaveOrderDetailsAsync` creates the `Order` aggregate/header from totals, addresses, payment result, checkout attributes, shipping fields, etc., then persists it through `_orderService.InsertOrderAsync(order)` and updates the custom order number. See [OrderProcessingService.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Libraries/Nop.Services/Orders/OrderProcessingService.cs:717). `OrderService.InsertOrderAsync` delegates to `_orderRepository.InsertAsync(order)` in [OrderService.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Libraries/Nop.Services/Orders/OrderService.cs:357).

10. `MoveShoppingCartItemsToOrderItemsAsync` is the direct conversion. It loops `foreach (var sc in details.Cart)`, loads the product, calculates unit/subtotal/tax/discounts, formats attributes, builds a new `OrderItem` with `OrderId = order.Id`, `ProductId = product.Id`, `AttributesXml = sc.AttributesXml`, `Quantity = sc.Quantity`, rental dates, prices, discounts, and weight, then calls `_orderService.InsertOrderItemAsync(orderItem)`. See [OrderProcessingService.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Libraries/Nop.Services/Orders/OrderProcessingService.cs:1273). `OrderService.InsertOrderItemAsync` delegates to `_orderItemRepository.InsertAsync(orderItem)` in [OrderService.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Libraries/Nop.Services/Orders/OrderService.cs:748).

11. After each cart line becomes an order item, the service adjusts inventory and publishes `ShoppingCartItemMovedToOrderItemEvent(sc, orderItem)`. The event carries both the source `ShoppingCartItem` and created `OrderItem`; see [ShoppingCartItemMovedToOrderItemEvent.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Libraries/Nop.Core/Domain/Orders/ShoppingCartItemMovedToOrderItemEvent.cs:6).

12. Cart cleanup happens after all order items are inserted: `MoveShoppingCartItemsToOrderItemsAsync` calls `_shoppingCartService.ClearShoppingCartAsync(details.Customer, order.StoreId)`. That method reloads the shopping cart, deletes all rows with `_sciRepository.DeleteAsync(cart, publishEvent: false)`, publishes `ClearShoppingCartEvent(cart)`, and updates `Customer.HasShoppingCartItems`. See [OrderProcessingService.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Libraries/Nop.Services/Orders/OrderProcessingService.cs:1339) and [ShoppingCartService.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Libraries/Nop.Services/Orders/ShoppingCartService.cs:656).

DI confirms the concrete service implementations: `IOrderService -> OrderService`, `IOrderProcessingService -> OrderProcessingService`, `IOrderTotalCalculationService -> OrderTotalCalculationService`, and `IShoppingCartService -> ShoppingCartService` in [NopStartup.cs](/home/simon/repos/lorq-worktrees/base-default-graphify__neutral__cart-to-order-flow__r1/src/Presentation/Nop.Web.Framework/Infrastructure/NopStartup.cs:198).
